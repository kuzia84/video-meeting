# Ralph Loop - правила автономной работы

## Как брать Issues

- Читай title, body и критерий готовности
- Проверяй, что указанная ветка уже существует (если нет - создай)
- Работай только в этой ветке, не создавай новых

## Именование коммитов

- По правилам skill commit

## Экономия токенов

- Следуй разделу «Working efficiently (minimize context)» из корневого CLAUDE.md и скиллу `read`: не читай файл целиком — ищи символ через Grep, читай нужный диапазон через Read (`offset`/`limit`).
- Issue бери минимально: `gh issue list --json number,title` чтобы выбрать, `gh issue view <N>` для нужного — не тяни весь список с телами.
- Тесты по ходу работы гоняй точечно: только затронутый workspace (`-w @video-meetings/api`) и по паттерну (`npx jest <file>` / `-t 'name'`), а не весь `npm run test`. Полный прогон — один раз в конце (см. «Правила завершения»).
- Держи вывод инструментов компактным: `git diff --stat`, `git log --oneline -10`, `--silent` у тестов когда нужен только факт зелёного.
- CLAUDE.md подгружаются в контекст автоматически — не открывай их ради справки и не перечитывай то, что уже в контексте.

## Правила реализации

- Сначала тесты - потом реализация (TDD)
- После каждого финального изменения запускай тесты (точечно — по правилу экономии выше)
- Если тесты красные после 5 попыток - остановись и напиши в Issue комментарий с описанием проблемы

## Правила завершения

- Убедись, что все тесты зеленые
- Убедись, что все требования выполнены
- Если менял архитектуру (новый модуль, контроллер, маршрут, поле схемы, shared-тип) — обнови соответствующий CLAUDE.md в этом же изменении: правило «keep docs current» из самого CLAUDE.md
- Если добавил новый backend-модуль (`apps/api/src/<module>/`) или новую область фронта — **создай для неё вложенный CLAUDE.md и добавь строку-ссылку в родительский** (правило «CLAUDE.md hierarchy»). Хук `.claude/hooks/check-module-docs.sh` предупредит, если модуль остался без CLAUDE.md
- Запусти skill /code-review для код-ревью. В комментарии к Issue при закрытии явно отметь, что ревью пройдено, и что исправил, если были замечания — без этой отметки шаг непроверяем со стороны
- Закрой Issue
- Не создавай PR, это сделает Stop Hook
