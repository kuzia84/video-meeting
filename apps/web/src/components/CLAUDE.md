# Shared components (`src/components/`)

App-local detail for reusable components. Parent: `apps/web/CLAUDE.md`. Use these; don't hand-roll — most were extracted after copy-paste drifted.

- **`AppHeader` (`app-header.tsx`)** — the header every signed-in page wears (logo home-link, user chip, logout). The chip is a `UserAvatar` + display name (`name` when set, else email) wrapping a `NextLink` to `/profile` with `aria-label={`Профиль: ${displayName}`}` (the accessible name contains the visible name for WCAG 2.5.3 Label in Name). Reads from `useCurrentUser` (see `../lib/CLAUDE.md`), so a rename updates it live. With no user (loading, signed out, failed load) it's a bare logo + logout — the page around it owns auth outcomes, and the header must not race it.
- **`PageShell` (`page-shell.tsx`)** — the frame for a signed-in **content** page: `AppHeader` above a centered `max-w-3xl` main. Wrap new content pages in it. Bare loading/redirect spinners stay outside it; the home view inlines its own layout (pre-existing exception).
- **`UserAvatar` (`user-avatar.tsx`, client)** — what pages render for a user's avatar: the uploaded picture when there is one, else the `DefaultAvatar` letter circle. The image sits behind the JWT-guarded `GET /users/me/avatar`, so it can't be an `<img src>` to the endpoint (no token) — the bytes are fetched with the auth header (`fetchAvatarBlob`) and shown via an **object URL** (revoked on unmount/change), rendered with a plain `<img>` (a `blob:` URL is already in memory, so `next/image` can't optimize it — the `no-img-element` rule is disabled on that line). Keyed on `avatarUrl` (a fresh UUID per upload), so replacing the avatar re-loads with no reload; while it loads, or when there is none, the letter circle stands in. Decorative (`aria-hidden`, `alt=""`).
- **`DefaultAvatar` (`default-avatar.tsx`)** — the letter circle: `avatarInitial(name, email)` on the user's own colour (`findAvatarColorSolution` keyed by `UserProfile.avatarColor`), both from `@video-meetings/shared`. **Theme-aware with no JS**: it sets the solution's four colours as inline CSS vars (`--avatar-bg-light/-fg-light/-bg-dark/-fg-dark`) which `.avatar-default` in `globals.css` consumes, picking the dark variant under `[data-theme='dark']` — no first-render flash, both themes clear WCAG AA (verified in the shared palette's unit tests). Decorative (`aria-hidden`); the fallback inside `UserAvatar`, never rendered directly by pages. **The only web consumer of the shared runtime avatar helpers** — Next.js transpiles shared source, so importing runtime values here needs no build step (unlike the API).
- **`MeetingForm` (`meeting-form.tsx`)** — the meeting fields, shared by **creating and editing** (create-meeting view and the meeting page's edit mode both render it). It **owns all validation and per-field errors**, and that validation is **load-bearing**: `validationBehavior="aria"` doesn't block submission, so the `handleSubmit` checks run with empty/invalid fields (mutation-tested — delete them and e2e fail). Errors go into a `fieldErrors` record fed to `<Form validationErrors>` (each reason under its field); each field clears its own error in `onChange`. A double-submit is guarded by a synchronous `isSubmittingRef`. `datetime-local` → ISO conversion lives here. `onSubmit` rejecting is how a caller reports a failure, which the form shows above the fields. `children` is a slot between the fields and the buttons (the create page's file picker — files belong to creating, not editing, so they're not a prop).
- **`ConfirmDeleteDialog` (`confirm-delete-dialog.tsx`)** — both deletions (a meeting, a meeting file) go behind it; each is irreversible and takes bytes with it. The confirm button deliberately has **no** `slot="close"`: it stays open until the request settles, so a failure has somewhere to be reported instead of vanishing into a closed dialog that implies success (only Cancel closes). Give the trigger and the confirm **different** names — they are different actions.
- **`Logo` (`logo.tsx`)** — the MeetingBrain SVG mark, decorative (`aria-hidden`). Used by `AppHeader` and the auth pages.
