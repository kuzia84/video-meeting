# Управление встречами

**Дата:** 2026-07-16
**Статус:** draft

## Цель

Пользователю нужно вести свои встречи целиком из интерфейса: видеть их список, заводить новые, править данные и убирать ненужные вместе с их файлами. Сейчас встречу можно только создать запросом к API, а увидеть — три последние на главной; без этого продукт нельзя открыть человеку.

## Пользовательские сценарии

- **Пользователь:** владелец встреч — **Действие:** открывает главную страницу — **Результат:** видит список своих встреч с постраничной навигацией; если встреч ещё нет, видит пустое состояние с приглашением создать первую.
- **Пользователь:** владелец встреч — **Действие:** нажимает «Добавить встречу» — **Результат:** попадает на отдельную страницу с формой: название, описание, время начала, время окончания и выбор файлов встречи.
- **Пользователь:** владелец встречи — **Действие:** заполняет форму и отправляет её — **Результат:** встреча создаётся, выбранные файлы загружаются следом с показом прогресса, и он оказывается на странице этой встречи.
- **Пользователь:** владелец встречи — **Действие:** отправляет форму с ошибкой (пустое название, окончание раньше начала) — **Результат:** видит причину рядом с полем, встреча не создаётся.
- **Пользователь:** владелец встречи — **Действие:** открывает встречу из списка — **Результат:** видит её данные и все загруженные файлы.
- **Пользователь:** владелец встречи — **Действие:** правит название, описание или время встречи и сохраняет — **Результат:** изменения сохранены и видны при следующем открытии встречи.
- **Пользователь:** владелец встречи — **Действие:** удаляет отдельный файл встречи и подтверждает удаление — **Результат:** файл исчезает из списка, перестаёт скачиваться и освобождает место на диске.
- **Пользователь:** владелец встречи — **Действие:** удаляет встречу и подтверждает удаление — **Результат:** встреча исчезает из списка вместе со всеми своими файлами, и место на диске освобождается без чьего-либо ручного вмешательства.
- **Пользователь:** владелец встречи — **Действие:** в диалоге подтверждения нажимает «Отмена» — **Результат:** ничего не удаляется.
- **Пользователь:** авторизованный пользователь, не владелец — **Действие:** пытается открыть, изменить или удалить чужую встречу либо её файл — **Результат:** доступа не получает.

## Скоуп

**В скоупе:**

- Главная страница как список встреч пользователя с постраничной навигацией и пустым состоянием.
- Кнопка перехода к созданию встречи с главной.
- Отдельная страница создания встречи: название, описание, время начала, время окончания, выбор файлов.
- Валидация формы с показом причины: обязательное название, окончание позже начала.
- Загрузка выбранных файлов вторым шагом, после создания встречи, с показом прогресса.
- Страница отдельной встречи: её данные и список загруженных файлов.
- Редактирование названия, описания и времени встречи.
- Удаление отдельного файла встречи владельцем, с подтверждением.
- Удаление встречи владельцем, с подтверждением.
- Автоматическое удаление всех файлов встречи — и записей о них, и самих файлов с диска — при удалении встречи.
- Поддержка редактирования и удаления на стороне API: сейчас их нет.
- Разграничение доступа: смотреть, менять и удалять встречу и её файлы может только владелец.
- E2E-тесты на все сценарии: API-уровень для создания, правки и удаления, браузерный — для списка, форм и подтверждений.

**Не в скоупе:**

- Сама загрузка файлов — она описана в [docs/meeting-file-upload.md](./meeting-file-upload.md); здесь только точка входа в форме создания.
- Переименование загруженных файлов.
- Восстановление удалённой встречи или файла, корзина, мягкое удаление.
- Участники встречи и шаринг наружу — модели участников нет.
- Повторяющиеся встречи и напоминания.
- Транскрипция, ИИ-анализ и обработка содержимого файлов.
- Поиск, фильтрация и сортировка списка встреч — только постраничный вывод в текущем порядке.
- Массовые операции: удаление нескольких встреч или файлов разом.

## Технические ограничения

- Каскадное удаление в базе убирает только записи о файлах — про сами файлы на диске оно не знает. Уборка диска не появится сама собой и должна быть частью удаления.
- Файловая система не участвует в транзакции базы, поэтому удаление записей и файлов нельзя выполнить одной атомарной операцией. Из двух возможных отказов терпим только один: осиротевший файл на диске место занимает, но интерфейс не ломает, а вот запись без файла даёт пользователю строку, которая не скачивается. Порядок операций должен оставлять именно восстановимый отказ.
- Удаление одного файла и удаление встречи целиком чистят диск одинаково — это один механизм, а не два.
- Шаг загрузки файлов в форме создания зависит от готовности загрузки (фазы 1–3 плана [docs/meeting-file-upload-plan.md](./meeting-file-upload-plan.md)). Пока её нет, форма способна только создавать встречу.
- Файл можно загрузить только в существующую встречу, поэтому в форме создания загрузка идёт после создания, а не одновременно с ним. Отсюда промежуточное состояние: встреча уже есть, файлы ещё нет.
- У встречи есть только владелец; сущности участников не существует, поэтому любая модель доступа шире «только владелец» потребует отдельной работы.
- Время начала и окончания у встречи обязательные, и окончание должно быть позже начала — форма обязана это учитывать.
- Технические грабли загрузки собраны в [docs/meeting-file-upload-research.md](./meeting-file-upload-research.md).

## Критерии готовности

- [ ] Пользователь без встреч видит на главной пустое состояние с приглашением создать встречу, а не ошибку и не пустой экран.
- [ ] Пользователь с 25 встречами видит на главной первую страницу списка, а переход на следующую показывает следующую порцию.
- [ ] Кнопка «Добавить встречу» с главной открывает отдельную страницу с формой.
- [ ] Форма с пустым названием не отправляется и показывает причину у поля.
- [ ] Форма, где окончание раньше начала, не отправляется и показывает причину у поля.
- [ ] После отправки корректной формы без файлов пользователь оказывается на странице созданной встречи, и её название, описание и время совпадают с введёнными.
- [ ] После отправки формы с файлом встреча создана, виден прогресс загрузки, а по завершении файл присутствует в списке файлов встречи.
- [ ] Если загрузка файла падает после создания встречи, встреча существует и открывается, а пользователь видит сообщение о неудавшейся загрузке.
- [ ] Изменённые название, описание и время сохраняются и видны после перезагрузки страницы встречи.
- [ ] Удаление файла после подтверждения убирает его из списка, прямая ссылка на него больше не отдаёт содержимое, и файла нет на диске.
- [ ] Удаление встречи после подтверждения убирает её с главной, встреча больше не открывается, и ни одного её файла на диске не осталось.
- [ ] Нажатие «Отмена» в диалоге подтверждения не удаляет ни файл, ни встречу.
- [ ] Пользователь Б не может открыть, изменить или удалить встречу пользователя А и не может удалить её файл.
- [ ] Неавторизованный пользователь, открывая страницу встречи или страницу создания, попадает на страницу входа.
- [ ] API-e2e покрывают создание, редактирование, удаление встречи, удаление файла и попытки чужого доступа к каждой из этих операций.
- [ ] Браузерные e2e покрывают список с пагинацией и пустым состоянием, создание встречи через форму, её редактирование и оба диалога подтверждения.
- [ ] Юнит-тесты, API-e2e и браузерные e2e проходят зелёными на чистой базе.
