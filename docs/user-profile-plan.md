# План: Страница профиля пользователя

**PRD:** [docs/user-profile.md](user-profile.md)
**Дата:** 2026-07-17

## Фаза 1. Профиль: просмотр

**Цель:** У пользователя появляется страница профиля, на которой он видит свои данные.
**Затрагивает:** database / backend / frontend

**Задачи:**

- Миграция: добавить пользователю необязательное поле имени и ссылку на аватар; у существующих пользователей оба пустые
- Запрос «текущий пользователь» в модуле Users и защищённый эндпоинт профиля, отдающий имя, email и ссылку на аватар
- Страница профиля во фронте с клиентской проверкой авторизации, как на главной
- Показ на странице профиля: имя (пока не задано — email), email только для чтения

**Критерий готовности:** Авторизованный пользователь открывает страницу профиля и видит свой email; тот же адрес без токена открывает страницу входа.

## Фаза 2. Дефолтный аватар

**Цель:** У каждого пользователя появляется узнаваемый аватар — буква в его собственном цвете — ещё до всякой загрузки.
**Затрагивает:** database / backend / frontend

**Задачи:**

- Набор цветовых решений: у каждого своё имя, вариант для светлой темы и вариант для тёмной в одном оттенке; буква читается на фоне в обеих
- Миграция: добавить пользователю имя доставшегося ему цветового решения и назначить его всем существующим пользователям; новым — назначать при регистрации
- Отдавать имя цветового решения в эндпоинте профиля
- Выбор буквы: первая буква имени с пропуском не-букв; если букв нет — по тому же правилу из email; если нет и там — нейтральный знак. Работает на любом алфавите. Покрыть модульными тестами
- Круг с буквой на странице профиля, когда картинка не загружена

**Критерий готовности:** Пользователь без загруженного аватара видит на странице профиля круг со своей буквой в своём цвете; цвет тот же после перезагрузки и повторного входа, а в тёмной теме оттенок узнаётся и буква остаётся читаемой.

## Фаза 3. Редактирование имени

**Цель:** Пользователь задаёт своё имя — и продукт начинает обращаться к нему по имени, включая букву на аватаре.
**Затрагивает:** backend / frontend

**Задачи:**

- Команда обновления имени в модуле Users и эндпоинт правки профиля с валидацией имени
- Форма редактирования имени на странице профиля
- Показ имени и аватара в хедере рядом друг с другом; клик по аватару ведёт на страницу профиля
- Единый источник текущего пользователя во фронте, чтобы после сохранения имени и хедер, и буква аватара обновлялись без перезагрузки
- E2E: имя видно в хедере и переживает перезагрузку; буква сменилась на первую букву нового имени, цвет круга при этом не изменился

**Критерий готовности:** Сохранённое на странице профиля новое имя сразу появляется в хедере вместе с аватаром, где буква сменилась на первую букву этого имени, а цвет круга остался прежним.

## Фаза 4. Загрузка аватара: хранение и отдача

**Цель:** Картинка пользователя принимается, проверяется и лежит на диске, заменяя предыдущую.
**Затрагивает:** backend / инфраструктура

**Задачи:**

- Конфигурация загрузки для аватаров: один файл, лимит 5 МБ, allowlist JPEG/PNG/WebP
- Проверка фактического содержимого файла, а не только расширения и заявленного типа; отказ понятным сообщением
- Эндпоинт загрузки: записать байты, обновить ссылку у пользователя, затем удалить прежний файл — в этом порядке
- Эндпоинт отдачи аватара, доступный только авторизованным пользователям
- E2E: успешная загрузка каждого формата, отказ по размеру, отказ по формату, замена прежнего файла

**Критерий готовности:** E2E-набор аватара зелёный: JPEG, PNG и WebP до 5 МБ принимаются, PDF и файл больше 5 МБ отклоняются с сообщением и не трогают прежний аватар, повторная загрузка заменяет ссылку и стирает прежние байты с диска.

## Фаза 5. Аватар в интерфейсе

**Цель:** Пользователь загружает картинку со страницы профиля, и она вытесняет букву и в профиле, и в хедере — насовсем.
**Затрагивает:** frontend

**Задачи:**

- Выбор и загрузка файла на странице профиля
- Показ загруженной картинки вместо круга с буквой — в профиле и в хедере
- Сообщения об отказе: неподдерживаемый формат, превышен размер — прежний аватар остаётся на месте
- E2E: загруженная картинка появляется в профиле и в хедере без перезагрузки и остаётся картинкой после смены имени

**Критерий готовности:** Загруженная на странице профиля картинка заменяет круг с буквой в профиле и в хедере без перезагрузки, и последующая смена имени её не возвращает.

## Фаза 6. Смена пароля

**Цель:** Пользователь меняет пароль сам, подтвердив, что знает текущий.
**Затрагивает:** backend / frontend

**Задачи:**

- Команда смены пароля: сверить текущий пароль, захешировать новый, применить те же требования, что при регистрации
- Защищённый эндпоинт смены пароля
- Форма на странице профиля: текущий пароль, новый, подтверждение нового
- Сообщения об отказе: неверный текущий пароль, новый не удовлетворяет требованиям, подтверждение не совпадает
- E2E: после смены вход с новым паролем проходит, со старым — нет

**Критерий готовности:** E2E-набор смены пароля зелёный: с верным текущим паролем смена проходит и вход с новым паролем работает, а старый пароль, неверный текущий, слабый новый и несовпадающее подтверждение отклоняются с сообщением.
